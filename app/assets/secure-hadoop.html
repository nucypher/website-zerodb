Secure Hadoop with DASHR

Delegated Access System for Hadoop based on Re-encryption

A cryptographically-enforced access control system for Hadoop.

Data at-­rest is always encrypted and encryption keys do not need to be shared between different data sources. The encryption/decryption overhead is minimal, particularly for jobs with intensive map phases. In DASHR, access tokens work like temporary encryption keys. Users are able to decrypt permitted objects only when their temporary encryption key is active.

Improved Key Management

With DASHR, Hadoop jobs don't require communication between compute nodes and the KMS, removing latency bottlenecks. Most (slow) requests to key management system are replaced by the efficient mathematics of encrypted-to-encrypted transformations.

Blazingly Fast, Scalable Key Rotation

DASHR executes secure key rotation in seconds instead of hours. Unlike TDE, it doesn’t require shutting down encryption zones or shuttling data in and out of the KMS.

Works with All Hadoop Distros

DASHR plugs directly into and extends Hadoop's Transparent Data Encryption (TDE). It operates at the HDFS layer and works alongside your existing Cloudera or Hortonworks deployments without hindering performance or user experience.
